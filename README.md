# 🌌 Asteroid Logistics: Проектная документация

🛸 Концепция игры: Asteroid Logistics
Жанр: Sci-Fi Idle-Clicker / Экономический симулятор. Платформы: Web (Desktop & Mobile). Сеттинг: Далекое будущее, промышленное освоение глубокого космоса.

1. Основная идея (Core Loop)
Игрок выступает в роли диспетчера орбитальной добывающей станции. Цель — построить максимально эффективную систему добычи и логистики, превращая астероиды в межгалактическую валюту (Кредиты).

Цикл игры:

Добыча: Дроны автоматически летают к астероидам и приносят ресурсы на склад.
Активность: Игрок может вручную разбивать астероиды кликами, получая ресурсы мгновенно.
Логистика: При наполнении склада вызывается транспортный корабль, который забирает груз и конвертирует его в деньги.
Развитие: На заработанные деньги покупаются новые дроны и улучшаются модули базы.
2. Игровые механики
💎 Система ресурсов и астероидов
В игре реализована система прогрессивной ценности ресурсов. Каждый астероид имеет визуальные отличия и требует разного количества взаимодействий:

Металл (Metal): Обычный ресурс (60% шанс появления). Цена: 1 CR. Плотность: 1 удар.
Космический лед (Ice): Средний ресурс (25% шанс). Цена: 5 CR. Плотность: 2 удара.
Кристаллы (Crystal): Редкий ресурс (10% шанс). Цена: 25 CR. Плотность: 3 удара.
Иридий (Iridium): Легендарный ресурс (5% шанс). Цена: 100 CR. Плотность: 3 удара.
Механика «Замирания»: При первом клике по летящему астероиду он замедляется, имитируя захват или прицеливание, что облегчает ручную добычу.

🤖 Флот дронов
Дроны — основа автоматизации. Игрок может комбинировать разные типы для оптимального баланса:

Basic: Универсальный юнит начального уровня.
Scout: Быстрый дрон для частых рейсов за мелкими порциями.
Heavy: Медленный гигант для перевозки больших объемов за один раз.
🏗️ Улучшения базы (HQ)
База имеет 3 визуальных уровня прогрессии (от простого круга до сложного неонового гексагона).

Рефайнерия: Повышает эффективность переработки (множитель цены продажи).
Грузовой отсек: Увеличивает лимит склада.
Ангар: Позволяет расширять количество активных дронов.
Автоматизация: Модуль, позволяющий базе самой отправлять транспорт при заполнении склада на 80%.
⚡ Система «Overclock» (Разгон)
Активная способность, временно (на 10 сек) ускоряющая работу всех систем. Визуально отображается через пульсацию центрального реактора базы.

3. UI/UX и Визуальный стиль
Визуальные принципы
Стиль «Blueprint»: Объекты (база, дроны) имеют строгие геометрические формы с неоновыми контурами.
Параллакс космоса: Звезды движутся с разной скоростью, создавая эффект глубины.
Информативность: HUD спроектирован так, чтобы игрок видел заполнение склада по типам ресурсов (цветовая индикация) без открытия меню.
Тактильность (Juiciness): Вылетающие цифры прибыли, вибрация экрана при продаже, реактивные струи двигателей.
Камера
Свобода обзора: Игрок может приближать камеру (Scroll/Pinch), чтобы рассмотреть детали дронов, или отдалять для обзора всего сектора добычи.
Панорамирование: В режиме зума доступно перемещение по сцене.
4. Системы удержания и прогресса
Оффлайн-прогресс: Игра рассчитывает добычу за время отсутствия игрока (до 8 часов), позволяя чувствовать прогресс даже без активной игры.
Архив (Энциклопедия): Система коллекционирования. Ресурсы скрыты «туманом войны», пока игрок не найдет их впервые. Каждое открытие сопровождается sci-fi уведомлением.
Масштабируемая сложность: Цена каждого следующего улучшения растет экспоненциально, требуя от игрока стратегического выбора между покупкой нового дрона или апгрейдом старых систем.
5. Техническая реализация (кратко)
Движок: Собственное решение на React requestAnimationFrame.
Состояния: Глобальный стор Zustand с полной синхронизацией в LocalStorage.
Траектории: Математический расчет полетов по кривым Безье, что делает движение «органичным».

## 🛠️ 2. Техническое описание (Tech Spec)

### Технологический стек
- **Frontend:** React + TypeScript.
- **Сборка:** Vite.
- **Состояние (State):** Zustand + middleware Persist (LocalStorage).
- **Стилизация:** Tailwind CSS + Lucide Icons.
- **Анимации:** CSS Keyframes + Framer Motion (частично).

### Архитектура систем
1.  **Game Engine:** Основной цикл (`requestAnimationFrame`), расчет дельты времени (delta time).
2.  **Resource System:** Динамическая генерация астероидов с разными типами ресурсов, редкостью и ценой.
3.  **Drone System:** Имитация полетов дронов по кривым Безье с учетом состояний (вылет, добыча, возврат).
4.  **Transport System:** Логика продажи ресурсов, расчет стоимости с учетом множителей рефайнерии.
5.  **Offline System:** Расчет добычи за время отсутствия игрока при входе в игру.

## 🧱 4. Детали реализации (Technical Deep Dive)

### Состояние (Zustand Store)
- **Store Structure:** Единый стор `useGameStore`, содержащий все данные: кредиты, ресурсы, дроны, транспорт, уведомления и множители.
- **Persistence:** Использование `persist` middleware для сериализации состояния в `asteroid-logistics-save`. Частичное сохранение (`partialize`) только необходимых данных (без временных уведомлений).
- **Time Scaling:** Все расчеты (полет дронов, таймеры буста) привязаны к `deltaTime` (мс), что обеспечивает плавность независимо от FPS.

### Визуализация полетов (Bezier Curves)
- Дроны и транспорт перемещаются по **квадратичным кривым Безье**.
- Каждому вылету присваиваются случайные параметры: `angle` (угол), `distance` (дальность) и `curveOffset` (кривизна), что создает ощущение живого флота.
- Отрисовка через `transform: translate(...) rotate(...)`, где угол поворота вычисляется динамически на основе траектории.

### Система ресурсов (Multi-resource Storage)
- Склад (`storage.current`) — это объект `Record<ResourceType, number>`.
- Логика заполнения учитывает суммарный объем всех ресурсов.
- При переполнении или оффлайн-добыче применяется **пропорциональное ограничение**, чтобы редкие ресурсы не вытеснялись обычным металлом.

### Управление и Camera System
- Реализована система координат в процентах (0-100) для объектов (астероиды, уведомления) и в пикселях для флота (относительно центра базы).
- **Zoom/Pan:** Обработка событий `wheel` (десктоп) и `touch` (пинч-зум на мобильных) с сохранением смещения (`offset`) для перемещения по сцене.
